================================
BACKEND REQUIREMENTS
================================

Use FastAPI + SQLAlchemy.

Vehicle model should include at minimum:
- id
- plate (required, indexed)
- property_name
- location
- tow_reason
- notes
- status
- created_at
- updated_at

Add audit/history support with separate tables or equivalent:
1. vehicle_status_history
2. vehicle_activity_log

Status values:
- Observed
- Warning Placed
- Marked for Tow
- Tow Requested
- Awaiting Tow Truck
- Towed
- Released
- Cancelled
- Cleared/Resolved

Required backend endpoints:

GET /health
- return healthy status

POST /vehicles/
- create vehicle

GET /vehicles/
- list vehicles
- optional filters:
  - plate
  - status
  - property_name

GET /vehicles/{vehicle_id}
- full detail including history/activity

GET /vehicles/search/{plate}
- search by plate

PATCH /vehicles/{vehicle_id}/status
- update status
- append status history
- append activity log

POST /vehicles/{vehicle_id}/notes
- add note
- append activity log

Backend requirements:
- proper imports
- proper package structure
- no duplicate FastAPI app definitions
- correct APIRouter registration
- CORS configured for Mini App frontend
- clear schema validation
- error responses that frontend and bot can handle cleanly
